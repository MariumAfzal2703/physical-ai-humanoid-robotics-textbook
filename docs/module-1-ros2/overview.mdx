Module 1 gives you a production-grade ROS 2 mental model for humanoid robotics. Instead of treating ROS 2 as a set of isolated commands, we treat it as a contract system: nodes publish typed state, subscribers consume it with timing guarantees, and launch files define reproducible runtime topology.

By the end of this module, you should be able to design a small but complete control stack: sensor ingestion, state estimation, command generation, and actuator output. You will also learn how to debug message timing and QoS mismatches early, which is one of the most common causes of integration failures in robotics teams.

```python
from dataclasses import dataclass

@dataclass
class JointState:
    name: str
    position: float
    velocity: float


def command_is_safe(position: float, min_pos: float = -1.2, max_pos: float = 1.2) -> bool:
    return min_pos <= position <= max_pos
```

## Key Takeaways

- ROS 2 is most effective when treated as a typed communication contract between subsystems.
- QoS and timing decisions are design decisions, not afterthoughts.
- Reproducible launch/config workflows dramatically reduce integration drift across machines.
